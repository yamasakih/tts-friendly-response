# TTS フレンドリーな応答ガイドライン

## 概要

Claude Code の応答は Style-Bert-VITS2 の TTS フックにより自動的に音声読み上げされる。日本語 TTS が英語のアルファベットを一文字ずつ読み上げてしまう問題を防ぐため、応答テキストを TTS に適した形式で記述する。

## ルール

### 英語の技術用語・変数名はカタカナで表記する

TTS は英語のアルファベット列を「エス、ティー、ワイ...」のように一文字ずつ読み上げてしまう。自然な読み上げのために、英語の技術用語や変数名はカタカナに変換して表記する。

**悪い例（TTS が不自然に読む）:**
- `style_weight` を 0.5 に変更しました
- `npm install` を実行してください
- `README.md` を確認してください

**良い例（TTS が自然に読む）:**
- スタイルウェイトを 0.5 に変更しました
- エヌピーエム インストールを実行してください
- リードミーを確認してください

### コードやコマンドは自然な日本語で要約する

コードブロック内のテキストは TTS により除去されるため、コードやコマンドの内容は本文中で自然な日本語として要約する。全文をそのまま読み上げず、何をするものかだけを伝える。

**悪い例:**
- `~/.claude/sbv2_venv/bin/python .claude/hooks/sbv2_server.py --device mps` で起動してください

**良い例:**
- デバイスオプションにエムピーエスを指定してサーバーを起動してください

### 応答は簡潔にする

TTS の読み上げ上限は 300 文字。重要な情報を先頭に配置し、簡潔に伝える。
